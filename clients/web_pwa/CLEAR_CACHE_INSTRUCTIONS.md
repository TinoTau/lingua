# 清除浏览器缓存说明

## 问题
如果前端按钮仍然显示"开始录音"而不是"开始转录"，可能是浏览器缓存了旧版本的文件。

## 解决方法

### 方法 1：硬刷新（推荐）
- **Windows/Linux**: `Ctrl + Shift + R` 或 `Ctrl + F5`
- **Mac**: `Cmd + Shift + R`

### 方法 2：清除浏览器缓存
1. 打开浏览器开发者工具（F12）
2. 右键点击刷新按钮
3. 选择"清空缓存并硬性重新加载"

### 方法 3：手动清除缓存
1. Chrome/Edge: 设置 → 隐私和安全 → 清除浏览数据 → 选择"缓存的图片和文件"
2. Firefox: 设置 → 隐私与安全 → Cookie 和网站数据 → 清除数据

### 方法 4：使用无痕模式
打开无痕/隐私模式窗口，访问 `index_realtime.html`

## 确认使用的是正确的文件

**重要**：请确保访问的是 **`index_realtime.html`**，而不是 `index.html` 或 `index_asr_only.html`。

正确的 URL：
- `http://localhost:8080/index_realtime.html` ✅
- `http://localhost:8080/index.html` ❌（批量处理模式）
- `http://localhost:8080/index_asr_only.html` ❌（仅 ASR 模式）

## 验证连续模式是否工作

1. 打开浏览器控制台（F12）
2. 点击"开始转录"
3. 开始说话
4. 观察控制台日志：
   - 应该看到持续的音频帧发送日志
   - 当检测到语音边界时，应该看到"📝 转录"和"🌐 翻译"日志
   - 音频应该立即播放

如果仍然没有连续输出，请检查：
- 后端服务是否正常运行
- WebSocket 连接是否建立成功
- 控制台是否有错误信息
